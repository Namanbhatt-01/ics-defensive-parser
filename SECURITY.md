# Security Policy

## Supported Versions

Only the latest release of this project is actively supported with security updates.

| Version | Supported |
| ------- | --------- |
| >= 1.0.0| Yes       |
| < 1.0.0 | No        |

## Reporting a Vulnerability

If you identify a security vulnerability within this project (such as a parsing crash, validation bypass, or logic leak), please do not open a public issue. Instead, report it through the following process:

1. Send an email to the project maintainer outlining the vulnerability detail.
2. Include the following details in your report:
   * **Description**: A clear description of the vulnerability.
   * **Impact**: How the vulnerability affects the runtime safety of the passive auditing engine.
   * **Proof of Concept (PoC)**: Sample mock log files or payloads that trigger the anomaly.
3. The maintainer will acknowledge receipt of the report within 48 hours and coordinate a patch within a reasonable timeline.
